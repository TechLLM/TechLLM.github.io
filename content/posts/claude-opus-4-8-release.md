## 왜 Opus 4.8인가

Anthropic은 오늘 Claude Opus 4.8을 공식 발표했다. Opus 4.7의 직계 후속 모델로, 같은 가격을 유지하면서 코딩·에이전트 작업·추론·실무 지식 작업 전반에서 성능을 끌어올렸다.

초기 테스터들은 Opus 4.8이 에이전트 작업에서 더 신뢰할 수 있고, 판단력이 날카로워졌다고 평가한다. Cursor의 공동창업자 Michael Truell은 "CursorBench에서 이전 Opus 모델을 모든 노력 수준에서 초과했으며, 도구 호출이 의미 있게 더 효율적"이라고 밝혔다.

## 주요 벤치마크 결과

가장 눈에 띄는 수치는 **Online-Mind2Web 84%** — 컴퓨터 사용 및 브라우저 에이전트 벤치마크에서 Opus 4.7과 GPT-5.5를 앞선 결과다.

Super-Agent 벤치마크에서는 Opus 4.8이 모든 케이스를 엔드투엔드로 완료한 유일한 모델이었으며, 비용 기준으로 GPT-5.5와 동등 수준에서 이를 달성했다.

법률 분야에서는 Legal Agent Benchmark 역대 최고점을 기록했고, all-pass 기준 10% 돌파에 성공한 첫 번째 모델이 됐다.

## 정직성 향상 — 4배 개선

Opus 4.8에서 가장 두드러진 변화 중 하나는 **정직성**이다. AI 모델의 고질적인 문제인 "근거가 부족한데도 자신감 있게 주장하는" 패턴이 대폭 줄었다.

Anthropic의 평가에 따르면 Opus 4.8은 자신이 작성한 코드의 결함을 그냥 통과시킬 확률이 이전 모델 대비 약 4배 낮다. 불확실성을 먼저 알리고, 입출력 문제를 선제적으로 플래그하는 경향이 강해졌다.

정렬(Alignment) 평가에서는 사용자 자율성 지원과 사용자 이익 추구 등 친사회적 지표에서 새로운 최고치를 기록했고, 기만이나 오용 협조 같은 비정렬 행동 비율은 Opus 4.7보다 현저히 낮았다.

## 다이나믹 워크플로: 코드베이스 전체를 한 번에

오늘 함께 발표된 **다이나믹 워크플로(Dynamic Workflows)**는 Claude Code의 판도를 바꾸는 기능이다.

단일 세션에서 수백 개의 병렬 서브에이전트를 실행할 수 있으며, Claude가 작업을 계획하고 실행한 뒤 출력 결과를 검증까지 자동으로 처리한다. 실질적인 적용 사례는 이렇다: 수십만 줄 규모의 코드베이스 마이그레이션을 킥오프부터 머지까지, 기존 테스트 스위트를 기준 삼아 자동 완료.

Enterprise·Team·Max 플랜의 Claude Code에서 리서치 프리뷰로 제공된다.

## 노력 제어 & API 업데이트

**노력 제어(Effort Control)**는 claude.ai와 Cowork에 새로 추가된 기능이다. 모델 선택기 옆에 위치하며 사용자가 응답 깊이를 직접 조절할 수 있다.

- **높은 노력**: 더 깊이 생각하고 더 나은 응답 제공
- **낮은 노력**: 빠른 응답, 사용량 제한 소비 절약

Opus 4.8은 기본값으로 높은 노력 수준에서 동작한다. 코딩 작업 기준 Opus 4.7의 기본 설정과 비슷한 토큰을 사용하면서 성능은 더 높다.

**Messages API** 업데이트도 주목할 만하다. 시스템 항목을 메시지 배열 안에서 직접 업데이트할 수 있게 됐다. 프롬프트 캐시를 유지하면서 에이전트 실행 중 권한·토큰 예산·환경 컨텍스트를 실시간 조정 가능해졌다.

## 실무자가 볼 핵심 포인트

- **Claude Code 사용자라면 다이나믹 워크플로 즉시 활성화**: 대형 리팩토링·마이그레이션 자동화에 직접 적용 가능 (Enterprise/Team/Max)
- **빠른 모드 비용 3배 인하**: 고속 응답이 필요한 프로덕션 워크로드 비용 구조 재검토 기회
- **노력 수준 조절로 사용량 최적화**: 단순 쿼리는 낮은 노력, 복잡한 분석은 높은/최대 노력으로 효율화
- **정직성 개선은 실무에서 중요**: 코드 리뷰·분석 작업에서 AI 결과물을 그대로 신뢰하기보다 검증 비용이 줄어든다
- **API mid-task 시스템 업데이트**: 장기 실행 에이전트에서 런타임 중 정책 변경이 가능해진 것은 엔터프라이즈 에이전트 아키텍처에 유용한 변화
