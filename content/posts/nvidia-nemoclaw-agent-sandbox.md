## 개요

AI 에이전트가 외부 서비스를 호출하고 코드를 실행하고 파일에 접근하는 능력이 커질수록, 그 에이전트가 어디에서 무엇을 할 수 있는지를 제어하는 일이 중요해진다. NVIDIA가 공개한 **NemoClaw**는 이 문제를 인프라 수준에서 다루는 도구다. AI 에이전트를 OpenShell 샌드박스 안에 격리해 실행하면서 네트워크 출력, 파일시스템 접근, 추론 요청을 모두 게이트웨이를 통해 제어한다.

## 핵심 요약

- NemoClaw는 호스트 CLI / 샌드박스 플러그인 / 버전 관리 블루프린트 3요소로 구성된다
- 에이전트는 OpenShell 샌드박스 안에서 실행되며 네트워크·파일·추론 접근이 모두 정책으로 통제된다
- 추론 요청은 샌드박스를 직접 빠져나가지 못하고 반드시 OpenShell 게이트웨이를 거친다
- 블루프린트는 불변·버전화·다이제스트 검증을 거쳐 공급망 안전을 보장한다
- `nemoclaw onboard` 명령 한 번으로 동일한 환경을 재현 가능하게 생성한다
- 에이전트가 허용되지 않은 호스트에 접근하면 차단 후 운영자 승인 UI가 뜬다

## 구조: 세 가지 핵심 요소

NemoClaw는 기존 OpenShell이나 에이전트 런타임을 대체하지 않는다. 이들을 반복 가능한 구조로 패키징한다. 세 요소는 분리되어 있고 각자 독립된 릴리스 주기를 가진다.

**호스트 CLI**는 온보딩, 프로바이더 검증, 설정 저장, OpenShell 명령 호출을 담당한다. 사용자와 운영자가 주로 상호작용하는 계층이다.

**샌드박스 플러그인**은 TypeScript 패키지로 OpenClaw 내부에서 실행된다. 관리형 추론 프로바이더 메타데이터를 등록하고, `/nemoclaw` 슬래시 커맨드와 런타임 컨텍스트 훅을 제공한다. 플러그인은 의도적으로 작게 유지된다.

**블루프린트**는 샌드박스 이미지, 정책, 추론 프로파일, 지원 에셋을 담은 버전 관리 YAML 패키지다. 실행 전 러너가 블루프린트를 불러와 다이제스트를 검증한다. 같은 블루프린트로 실행하면 항상 같은 환경이 만들어진다.

## 게이트웨이가 하는 일

NemoClaw의 핵심은 **OpenShell 게이트웨이**다. 이 게이트웨이는 NemoClaw 컨트롤, 샌드박스, 추론 프로바이더, 외부 인테그레이션 사이에 위치한다.

에이전트가 모델 API를 호출하면 그 요청은 샌드박스를 직접 빠져나가지 못한다. OpenShell이 모든 추론 호출을 가로채 설정된 프로바이더로 라우팅한다. 샌드박스 안에서는 `inference.local`과 통신하고, 실제 프로바이더 크리덴셜과 엔드포인트는 호스트가 소유한다. NVIDIA 엔드포인트, NIM, 또는 호환 API를 프로바이더로 선택할 수 있으며, Model Router 프로바이더를 선택하면 요청마다 NVIDIA 모델 풀에서 최적 모델을 선택한다.

## 보호 레이어

샌드박스는 기본 정책으로 4개 레이어의 제어를 적용한다.

| 레이어 | 보호 내용 | 적용 시점 |
|--------|-----------|-----------|
| 네트워크 | 허가되지 않은 외부 연결 차단 | 런타임 핫 리로드 가능 |
| 파일시스템 | 시스템 경로 읽기 전용; `/sandbox`·`/tmp`만 쓰기 허용 | 샌드박스 생성 시 고정 |
| 프로세스 | 권한 상승 및 위험한 시스템 콜 차단 | 샌드박스 생성 시 고정 |
| 추론 | 모델 API 호출을 제어된 백엔드로 재라우팅 | 런타임 핫 리로드 가능 |

에이전트가 허용 목록에 없는 호스트에 접근하려 하면 OpenShell이 요청을 차단하고 TUI에 운영자 승인 요청을 띄운다. 승인된 엔드포인트는 현재 세션에서는 유지되지만 기본 정책 파일에 저장되지는 않는다.

## 설계 원칙

NemoClaw 아키텍처는 다섯 가지 원칙을 따른다.

**얇은 플러그인, 버전 관리 블루프린트:** 샌드박스 플러그인은 작고 안정적으로 유지하고, 호스트 측 오케스트레이션이 자체 릴리스 주기로 발전한다.

**CLI 경계 존중:** `nemoclaw` CLI가 샌드박스 관리의 주 인터페이스다.

**공급망 안전:** 블루프린트 아티팩트는 불변이고 버전화되며, 실행 전 다이제스트 검증을 거친다.

**OpenShell 기반 라이프사이클:** NemoClaw는 OpenShell 리소스를 내부에서 오케스트레이션하며, `nemoclaw onboard`가 운영자 진입점이다.

**재현 가능한 설정:** 동일한 블루프린트와 정책 정의로 실행하면 항상 동일한 샌드박스가 만들어진다.

## 실무자가 볼 핵심 포인트

NemoClaw는 "에이전트를 실서비스에 안전하게 배포하려면 무엇이 필요한가"에 대한 NVIDIA의 답변이다.

현재 많은 팀이 AI 에이전트의 실서비스 배포를 앞두고 가장 고민하는 지점이 외부 API 호출 제어, 크리덴셜 관리, 예상치 못한 파일시스템 접근이다. NemoClaw는 이 세 문제를 OpenShell 게이트웨이라는 단일 제어 지점으로 해결한다.

주목할 부분은 추론 라우팅의 투명성이다. 에이전트 코드는 `inference.local`과만 통신하고 실제 엔드포인트나 크리덴셜을 알 필요가 없다. 이는 에이전트 코드 수정 없이 프로바이더를 교체하거나 모델 라우팅 정책을 바꿀 수 있다는 뜻이다.

블루프린트 기반의 재현 가능한 환경은 CI/CD 파이프라인에서 에이전트를 테스트하거나 여러 환경에 동일한 에이전트를 배포할 때 실질적인 이점이 된다.
