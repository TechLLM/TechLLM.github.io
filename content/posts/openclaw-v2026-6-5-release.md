## 개요

OpenClaw v2026.6.5가 공개됐습니다. 새 기능을 잔뜩 박아 넣은 릴리스가 아니라, 그동안 손볼 곳이 쌓여 있던 모서리를 한꺼번에 다듬은 안정화 위주 업데이트입니다. QQBot 답변에 섞여 나오던 모델의 추론 태그가 사라졌고, MCP 도구가 돌려주는 결과의 형식이 어긋나도 세션이 깨지지 않습니다. 한동안 말썽이던 Google Vertex 카탈로그도 다시 정상 동작합니다.

릴리스 노트만 보면 잔손질처럼 보이지만, 실제로 운영팀이 겪던 알람을 줄여주는 변경이 많습니다.

## 핵심 요약

- **QQBot 답변 정리**: 모델의 thinking 태그가 그대로 채널로 흘러나가던 문제 차단
- **MCP 결과 강제 정규화**: 텍스트·이미지가 아닌 콘텐츠 블록을 materialization 경계에서 정리해 Anthropic API 오류와 깨진 세션 기록 방지
- **Vertex·Ollama 카탈로그 복구**: ADC 인증과 Ollama 메타데이터, Gemini 경로 prefix 처리 정상화
- **Matrix 음성 메시지 사전 점검**: 보내기 전 검증을 추가하고, 스레드 기준의 읽기·답장 동작 강화
- **저장소 SQLite 전환**: 인증 프로파일, Matrix 동기화, 메모리 위키 가져오기, 샌드박스 레지스트리를 SQLite로 옮겨 내구성 확보
- **doctor·cron 마이그레이션**: 레거시 cron JSON을 doctor preflight에서 자동 이관해 업그레이드 시 끊김 방지

## 본문

### QQBot — 모델의 속생각이 채널에 노출되던 문제

QQBot 답변에서 thinking 같은 추론 태그가 그대로 새어 나오던 버그가 잡혔습니다. extended thinking을 켜고 운용 중인 채널에서는 사용자에게 그대로 노출되는 게 가장 큰 불만이었는데, 이제 응답 단계에서 한 번 더 정리해 보내줍니다. 같은 맥락에서 extended-thinking 세션이 prompt cache 만료나 Gateway 재시작을 만나도 다시 살아나도록 스트림 이벤트 처리도 손봤습니다. message_start가 도착해야 복구 재시도가 트리거되고, 컴팩션 뒤 남은 thinking 서명도 다시 보내기 전에 미리 떨어뜨립니다.

### MCP 도구 결과 — Anthropic 형식 어긋남으로 깨지던 세션

이번 릴리스에서 가장 가려운 곳을 긁어준 부분이 MCP입니다. 일부 MCP 도구가 텍스트나 이미지가 아닌 블록(예: resource embed)을 돌려주면 Anthropic API가 형식 오류를 뱉었습니다. 한 번 난 오류는 세션 기록에 그대로 남아서, 재실행 때마다 같은 오류가 또 났습니다. 이제 materialization 경계에서 비표준 블록을 텍스트로 한 번 정리합니다. 도구 쪽은 자유롭게 결과를 던지고, 모델에는 안전한 형식으로만 들어갑니다.

여기에 MCP 임대 타임스탬프, prompt cache 도구 이름, 프로바이더 카탈로그 검증도 더 엄격해졌습니다. 잘못된 도구 이름 때문에 cache miss가 누적되는 패턴이 사라집니다.

### Google Vertex — ADC 사용자 카탈로그 복구

ADC 인증으로 Vertex를 쓰는 환경에서 정적 카탈로그가 비어 보이던 문제가 풀렸습니다. 생성된 카탈로그에 ADC 인증 마커가 그대로 살아남고, 런타임 모델 해석도 정상 동작으로 돌아왔습니다. 그동안 "Vertex를 붙였는데 모델 목록이 안 뜬다"는 제보가 종종 올라왔는데, 이번 패치로 정리됩니다.

Ollama 카탈로그도 메타데이터 보존이 정상화됐고, Google 프로바이더 prefix가 Gemini 경로 안에 중복으로 들어가던 문제도 정리됐습니다.

### Matrix — 음성 메시지 preflight과 스레드 인식

Matrix 채널에서 음성 메모(voice note) 전송 전에 사전 점검을 한 번 거칩니다. 매개변수가 맞지 않거나 채널 권한이 부족한 상태에서 메시지를 던지고 실패 로그만 쌓이던 패턴이 줄어듭니다.

스레드 인식도 들어갔습니다. 읽기와 답장이 스레드 기준으로 동작해, 채널 메인 흐름과 스레드 내부 대화가 섞이지 않습니다. QA 시나리오까지 같이 정리돼서 회귀 검증이 쉬워졌습니다.

### Parallel — 새 웹 검색 프로바이더

웹 검색 쪽엔 Parallel이 정식 프로바이더로 들어왔습니다. API 키 자동 탐지와 엔드포인트 처리, 라이브 프로바이더 테스트, 문서까지 함께 묶여 있어, 키만 꽂으면 바로 검색 결과를 받을 수 있습니다. 기존 검색 도구와 병렬 호출이 가능해, 여러 검색 소스를 합쳐 쓰는 워크플로에 잘 맞습니다.

### 채널별 잔손질 — WhatsApp·Feishu·Mattermost·iMessage

- **WhatsApp**: 계정별 설정이 바뀌면 재시작이 잡힌 시간 안에 끝나도록 부팅 대기 시간에 상한이 들어갔습니다.
- **Feishu**: 일시적인 rate limit 오류는 선형 백오프로 재시도합니다.
- **Mattermost**: 슬래시 커맨드 상태를 유지하고, 기본 답장은 이미 열린 스레드 안으로 들어갑니다.
- **iMessage**: 실패와 타임아웃 메시지가 사용자가 알아볼 수 있도록 정리됐습니다.

채널 운영을 해보면 가장 자주 들어오는 민원이 "왜 답이 안 가요"인데, 이번 잔손질 대부분이 그 직접적인 원인을 줄이는 쪽입니다.

### 저장소 — SQLite 전환으로 잡은 내구성

이번 릴리스의 숨은 핵심은 저장소 정리입니다. Matrix 동기화 상태, 메모리 위키 가져오기, 샌드박스 레지스트리, 인증 프로파일이 한꺼번에 SQLite로 옮겨졌습니다. JSON 파일을 그대로 덮어쓰던 방식에서는 동시 쓰기가 겹칠 때 파일이 망가지는 일이 있었는데, 그 통로가 막혔습니다.

플러그인 핀(pin)도 강해졌습니다. ClawHub에서 받는 스킬은 GitHub 저장소에 고정된 커밋을 박아두는 방식으로 설치돼, 같은 버전이 항상 같은 코드를 받습니다.

### 보안 잔손질

- HTTP 도구 중 소유자 전용 권한이 필요한 항목은 게이트가 한 번 더 들어옵니다.
- MCP HTTP 리다이렉트가 외부 호스트로 임의로 빠지지 않도록 보호 단계가 추가됐습니다.
- 인라인 이미지 페이로드 redaction이 data URL과 복구된 트랜스크립트 이미지까지 다 잡습니다.

### 세션·트랜스크립트 안정성

stale history를 다시 불러와도 사용자 메시지가 낙관적으로 유지되고, runId가 재할당돼도 흐트러지지 않습니다. 세션 트랜스크립트 재작성 시 레지스트리 마커도 일관되게 따라옵니다. 운영하다 보면 가끔 트랜스크립트가 어긋나면서 검색이 안 되는 케이스가 있었는데, 이 부분도 같이 막혔습니다.

### 업그레이드 안전망

doctor의 config preflight가 레거시 cron JSON 저장소를 런타임이 읽기 전에 자동으로 마이그레이션합니다. 그동안 버전을 건너뛰어 올린 환경에서 cron 작업이 사라져 보이는 일이 있었는데, 이번 릴리스로 안전망이 생겼습니다. 서비스 환경 변수도 함께 정리됐습니다.

### 릴리스 번호 — YYYY.M.PATCH

이번부터 릴리스 번호 체계가 `YYYY.M.PATCH` 월간 형식으로 바뀝니다. v2026.6.5는 2026년 6월의 다섯 번째 패치라는 뜻입니다. 패치가 자주 나오는 워크스페이스 운영 특성에 맞춰, 어떤 달의 릴리스인지 한눈에 보이도록 한 변경입니다.

## 실무자가 볼 핵심 포인트

운영 관점에서 체감이 가장 큰 건 두 가지입니다. 하나는 MCP 결과 정규화입니다. 비표준 블록 때문에 세션이 깨지던 패턴이 알아서 보정되니, 새로 만든 MCP 도구를 붙일 때 형식을 빈틈없이 맞추지 않아도 운영이 멈추지 않습니다. 다른 하나는 SQLite 전환입니다. 동시 쓰기로 파일이 망가지는 알람이 줄어듭니다.

ADC만으로 Vertex를 쓰던 팀은 이번 업그레이드로 모델 목록이 다시 차오르는지 꼭 확인해 보세요. Matrix를 비즈니스 채널로 쓰는 팀은 스레드 기준 동작이 기본값이 되므로, 봇 응답이 갑자기 스레드 안쪽으로 들어가는 변화에 적응해야 합니다.

ClawHub 핀 강화와 도구 검증 강화 덕분에 플러그인 운영 시 예측 가능성도 함께 올라갑니다. 패치 단계라도 SQLite 마이그레이션이 포함된 만큼, 업그레이드 전 백업은 평소보다 한 번 더 챙기는 게 안전합니다.
