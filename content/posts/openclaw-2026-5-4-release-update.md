# OpenClaw 2026.5.4: Google Meet 음성 에이전트가 달라졌다

2026년 5월 5일, OpenClaw 버전 2026.5.4가 공개됐다. 이번 릴리스는 단순한 버그 수정 묶음이 아니다. **Google Meet와 Voice Call 환경에서의 실시간 음성 에이전트 경험**을 근본적으로 개선했고, Gateway 스타트업 성능, 플러그인 마이그레이션 힌트, Control UI 반응성, 주요 채널 안정성까지 광범위한 변화가 담겼다.

릴리스 규모 자체가 크다. Changes 항목만 80개, Fixes 항목은 100개를 훌쩍 넘는다. 핵심 영역별로 뜯어보자.

---

## 하이라이트: Google Meet Twilio 다이얼인 × Gemini 음성 브리지

이번 릴리스의 가장 큰 변화는 단 하나의 PR(#77064)에서 나왔다.

**Twilio 다이얼인으로 Google Meet에 참가하는 OpenClaw 음성 에이전트가 Gemini 실시간 음성 브리지를 통해 응답하도록 바뀌었다.** 구체적으로 어떤 변화인지 살펴보면:

- **페이싱된 오디오 스트리밍**: 오디오를 일정 속도로 스트리밍해 지터 없이 안정적인 음성 재생을 보장한다.
- **백프레셔 인식 버퍼링**: Twilio 오디오 큐에 상한을 두고, 과부하 시 실시간 스트림을 먼저 닫아 WebSocket 백프레셔 뒤로 오디오가 쌓이는 현상을 방지한다.
- **끼어들기(Barge-in) 큐 초기화**: 참가자가 말을 시작하면 대기 중인 에이전트 발화 큐를 즉시 비운다. 에이전트가 자신의 목소리를 새 발화로 오인해 루프에 빠지는 문제도 함께 해결했다.
- **TwiML 폴백 제거**: 실시간 음성 중에는 TwiML 폴백을 사용하지 않아, Meet 참가자 입장에서 훨씬 자연스러운 응답 속도를 경험할 수 있게 됐다.

기존에는 Twilio 다이얼인 경로가 실시간 Gemini 브리지를 제대로 타지 못했다. 이제 Meet 회의에서 OpenClaw 음성 에이전트를 불러 쓰는 시나리오가 실용적인 수준으로 올라왔다고 봐야 한다.

### Google Meet 전반의 추가 개선

하이라이트 외에도 Google Meet 관련 변경이 이번 릴리스 전체에 걸쳐 산재해 있다:

- `realtime.introMessage: ""`를 존중해 실시간 Chrome 참가 시 무음으로 시작할 수 있게 됐다.
- 기본 Chrome 토크백 경로를 `mode: "agent"` (STT → OpenClaw 에이전트 → TTS)로 확립하고, 직접 양방향 모델은 `mode: "bidi"`로 명확히 분리했다.
- `realtime.strategy: "agent"`를 기본값으로 설정해 에이전트 기반 응답이 표준 경로가 됐다.
- Chrome 오디오 버퍼 기본값을 8192바이트에서 4096바이트로 낮춰 토크백 지연을 줄였다.
- 전사 공급자와 양방향 음성 공급자 설정을 분리해 Gemini Live가 직접 양방향 폴백을 지원하면서도 기본 OpenClaw 에이전트 경로를 방해하지 않도록 했다.
- 실시간 소켓이 준비 완료 전에 닫히는 경우, 타임아웃 오류로 잘못 분류하던 버그를 수정했다.

## Gateway 성능과 진단 개선

### 스타트업 성능 최적화

Gateway 시작 시간을 실질적으로 줄이는 작업이 여러 방향에서 진행됐다:

- 모델 카탈로그 테스트 헬퍼, 세션 조회 코드, QR 페어링 헬퍼, TypeBox 메모리 툴 스키마 생성 코드를 hot 스타트업 임포트 경로에서 제거했다.
- Native로 로드 가능한 플러그인 시작 경로에서 `jiti`를 임포트하지 않도록 해, 소스 변환 로더 비용을 실제로 필요한 경우에만 지불하게 됐다.
- 비readiness 사이드카를 ready 신호 이후로 연기하고, 채널 플러그인 배럴 임포트를 hot-path에서 제거하며, 신뢰된 번들 플러그인 메타데이터 fast-path를 추가했다.

### Windows 로컬호스트 바인딩 수정

Windows에서 libuv의 듀얼스택 `::1` 동작이 localhost HTTP 요청을 막는 문제가 수정됐다. 이제 기본 루프백 게이트웨이 리스너를 `127.0.0.1`에만 바인딩한다(#69701, #69674 수정).

### 진단 및 관찰 가능성

- 스타트업 페이즈 스팬, 활성 작업 레이블, 멈춘 터미널 브리지 마커, 기본 sync-I/O 추적을 `pnpm gateway:watch`에 추가해 느린 Gateway 턴을 로그에서 파악하기 쉬워졌다.
- Discord 전송 저하 및 게이트웨이 이벤트 루프 정체 신호를 `openclaw channels status`, `openclaw status --deep`, 페치 타임아웃 로그에 추가했다.

## 플러그인 성능 및 마이그레이션 힌트

### 콜드 스캔 최소화

플러그인 메타데이터 콜드 스캔을 피하는 작업이 에이전트와 플러그인 레이어 모두에서 이루어졌다:

- BTW, 컴팩션, 임베디드 실행 모델 생성, PDF 모델 설정에서 명시적인 에이전트 디렉터리 모델 갱신이 현재 워크스페이스 범위 플러그인 메타데이터 스냅샷을 재사용하도록 변경했다.
- Unscoped 모델 카탈로그와 매니페스트 계약 리더도 같은 스냅샷을 재사용하도록 했다.

### 마이그레이션 힌트 개선

업그레이드 후 `plugins.entries` 또는 `plugins.allow`가 미설치 공식 외부 플러그인을 참조할 때, 이제 `openclaw plugins install <spec>` 명령어를 명시적으로 안내한다. 기존에는 "유효한 플러그인 설정을 제거하라"는 오해를 살 수 있는 메시지가 나왔다.

## OpenAI/Codex 미디어 전사 개선

Codex 오디오 전사가 런타임 및 매니페스트 메타데이터에 정식으로 등록됐다. 이제 활성 Codex 채팅 모델은 오디오 전사 기본값으로 OpenAI 전사 경로를 사용한다. 기존에는 채팅 모델 ID가 오디오 전사 API로 그대로 전달되는 문제가 있었다.

Codex 관련 추가 수정:

- OAuth 진행 스피너를 수동 리디렉트 붙여넣기 프롬프트 이전에 정지시킨다.
- Codex 앱서버 커맨드 출력, 실패 응답, 승인 프롬프트를 채팅에 게시하기 전에 sanitize한다.
- 잘못된 형식의 `/codex` 컨트롤 커맨드 및 진단 확인에 대해 fail-closed 처리를 적용했다.

## Control UI 개선

이번 릴리스에서 Control UI도 꽤 많이 바뀌었다.

### 헤더 및 네비게이션

대시보드 브레드크럼에 현재 에이전트 이름이 표시된다. 세션 키까지 함께 표시하지 않아 상단바가 혼잡해지는 것을 방지했다.

### Cron 잡 UI

새 잡 사이드바를 접을 수 있게 만들었다. 잡 목록이 화면을 더 넓게 활용할 수 있고, 폼은 클릭 한 번이면 바로 열린다.

### 채팅 인터페이스

- 에이전트 우선 필터를 채팅 세션 피커에 추가했다.
- 전화기/태블릿/데스크톱 너비에서 채팅 컨트롤과 컴포저가 반응형으로 동작하도록 개선했다.
- 연속으로 반복되는 동일 텍스트 메시지를 카운트 배지가 달린 단일 버블로 접어, 하트비트 확인 메시지 같은 노이즈를 줄였다.
- 초기 채팅 로드 중 중복 아바타 갱신을 방지하고, 스크롤 다운 시 채팅 컨트롤 행을 숨긴다.

### 성능 및 기타

- 긴 애니메이션 프레임 또는 긴 태스크 항목을 디버그 이벤트 로그에 기록해 느린 대시보드 렌더를 추적하기 쉬워졌다. 다만 이 진단 데이터는 게이트웨이/플러그인 이벤트를 밀어내지 않도록 상한이 적용됐다.
- Control UI/Talk: 실패한 Talk 시작 오류를 dismiss 가능하게 만들었고, 다음 Talk 클릭 시 stale 세션을 자동 정리한다.
- 미디어 패치: 어시스턴트 미디어 페치에 단기 스코프 티켓을 발급해, 채팅 이미지 URL에 장기 인증 토큰이 노출되지 않도록 했다(#70830, #77097 수정).

## 채널별 개선

### Slack

- Block Kit 진행 초안을 위한 `streaming.progress.render: "rich"` 추가.
- Block Kit 한도 초과 시 최신 rich 진행 라인을 유지한다.
- 재개된 부모 `message.send` 호출이 원래 Slack 스레드에 유지되도록 수정.
- 스레드 참여 기록을 올바르게 갱신해 멘션 없이도 봇 참여 스레드에서 답장할 수 있게 됐다(#77648 수정).

### WhatsApp

- WhatsApp 채널/뉴스레터 `@newsletter` 아웃바운드 메시지 타겟 지원 추가(#13417 수정).
- 초기 및 재시작 소켓에서 로그인 QR 출력을 활성 런타임으로 라우팅하도록 수정.

### Telegram

- 플러그인 소유 포럼 토픽 타겟을 에이전트 메시지 툴에서 허용.
- 공유 인터랙티브 답장 버튼이 인라인 키보드로 렌더링되도록 수정(#76238).
- 이미지가 아닌 미디어 첨부파일이 `<media:image>`로 잘못 전달되던 버그 수정(#69793 수정).
- 멀티청크 답장에서 순간적인 추가 버블이 나타났다 사라지는 버그를 수정했다.

### Discord

- Discord 전송 저하 및 게이트웨이 이벤트 루프 정체 신호를 status 명령어에 추가.
- WSL2 같이 `/users/@me` REST가 느린 환경에서도 채널이 온라인으로 올라오도록 게이트웨이 모니터를 시작 프로브 완료 전에 시작한다(#77103, #77398, #77526 수정).
- Discord REST 및 게이트웨이 WebSocket 시작 경로에 IPv4를 우선 사용해 IPv4-only 네트워크에서 READY가 멈추는 문제를 수정했다.
- 최종 답장 전달 실패를 실패 턴으로 처리해, 메시지가 드롭됐을 때 완료로 표시되지 않도록 했다(#77520 수정).

## Provider 개선: OpenRouter

OpenRouter 관련 두 가지 opt-in 기능이 추가됐다:

1. **응답 캐싱**: 검증된 OpenRouter 경로에서만 `X-OpenRouter-Cache`, `X-OpenRouter-Cache-TTL`, 캐시 초기화 헤더를 전송한다.
2. **앱 어트리뷰션 카테고리 확장**: 코딩, 프로그래밍, 쓰기, 채팅, 개인 에이전트 용도를 OpenRouter에 광고한다.

또한 DeepSeek V4 `reasoning_effort` 값 처리도 수정됐다. `max`를 `xhigh`로 매핑하지 않던 버그로 인해 `openrouter/deepseek/deepseek-v4-pro`가 400 오류를 반환하던 문제가 해결됐다(#77350 수정).

## 보안 강화

이번 릴리스에는 눈에 띄는 보안 수정도 여럿 포함됐다:

- **exec 승인**: `env -S`, `exec`, `env -P` 같은 커맨드 캐리어를 통한 인라인 eval 페이로드 탐지를 강화했다.
- **Windows ACL 헬퍼 경로 검증**: `icacls.exe`/`whoami.exe` 해석 시 `SystemRoot`/`WINDIR` 환경 변수 조작을 통한 리다이렉션을 차단했다.
- **브라우저 SSRF**: 이미 선택된 탭에서 디버그/익스포트/읽기 경로가 데이터를 수집하기 전에 SSRF 정책을 적용한다.
- **WebSocket 인증 스코프**: 언바운드 WebSocket 인증 스코프를 클램프 처리해 범위를 제한했다.

## 업그레이드 시 주의사항

- **Windows 환경**: POSIX `/tmp/openclaw` 경로 문제가 수정됐다. 이제 로그/TTS 임시 파일이 `%TEMP%\openclaw-<uid>`로 올바르게 저장된다(#60713 수정).
- **Docker macOS**: `OPENCLAW_CONFIG_DIR` 및 `OPENCLAW_WORKSPACE_DIR`이 컨테이너 내부에 핀됐다. 이전에는 호스트 경로가 런타임 코드로 누출되어 `EACCES: permission denied, mkdir '/Users'` 오류가 발생했다(#77436 수정).
- **`doctor --fix`**: 이번 릴리스에서 `doctor`가 여러 마이그레이션을 자동 처리한다. Google Meet 설정, 레거시 그룹 채팅 config, 플러그인 레지스트리 정리 등이 포함된다.
- **플러그인 CalVer 수정 버전**: `2026.5.3-1` 같은 수정 버전 빌드가 이제 기본 API 범위를 만족하는 것으로 처리된다.

## 정리

OpenClaw 2026.5.4는 Google Meet 음성 에이전트 경험을 중심으로 한 기능 개선과, 수십 개의 안정성·성능 수정이 뒤따르는 릴리스다. 대규모 배포 환경에서 매일 OpenClaw를 쓰는 팀이라면 Gateway 스타트업 최적화와 플러그인 콜드 스캔 감소, Discord/Telegram 안정성 수정이 체감될 가능성이 높다.

음성 통화 워크플로우를 구축 중이라면, 이번 릴리스를 기점으로 Twilio × Gemini 브리지 경로를 적극적으로 검토해볼 만하다.

전체 변경사항은 [공식 릴리스 페이지](https://github.com/openclaw/openclaw/releases/tag/v2026.5.4)에서 확인할 수 있다.
