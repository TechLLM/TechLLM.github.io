## 개요

ChatGPT가 한 걸음 더 '개인 비서'에 가까워졌습니다. OpenAI가 사이드바에 **'예약(Scheduled)'** 페이지를 새로 만들었습니다. 그동안 흩어져 있던 예약 작업을 한 화면에서 보고, 일시정지·수정·삭제까지 모두 처리할 수 있게 됐습니다. 리서치 작업은 웹과 연결된 앱을 알아서 뒤지다가 실제로 변화가 생긴 순간에만 알림을 보냅니다. 그리고 기존 'Pulse' 기능은 이번 업데이트로 사라지고 예약 작업 안으로 흡수됩니다.

---

## 핵심 요약

- **새 메뉴**: ChatGPT 사이드바에 'Scheduled' 페이지 신설, 활성 작업 한곳에서 관리
- **가능한 조작**: 작업 조회·일시정지·수정·삭제 전부 한 화면에서
- **리서치 자동화**: 웹과 연결된 앱을 주기적으로 확인해 **변화가 있을 때만** 알림
- **예약 옵션**: 특정 시간 또는 오전·오후·저녁 등 시간대 단위 지정 가능
- **실행 빈도**: 작업당 시간당 최대 1회, 사용자가 비활성 상태가 되면 자동 일시정지
- **이용 대상**: Plus, Pro, Business, Enterprise 사용자 (요금제별로 동시 작업 수 차등)
- **Pulse 통합 종료**: 기존 Pulse 기능은 예약 작업으로 흡수되며 단독 메뉴는 사라짐

## 본문

### 사이드바에 새로 생긴 '예약' 페이지

이번 업데이트의 핵심은 단순합니다. 흩어져 있던 예약 작업을 **한곳에 모아 보여주는 전용 페이지**를 만든 것입니다. 사이드바의 'Scheduled' 항목을 누르면 현재 동작 중인 모든 작업이 줄지어 뜹니다.

이전에는 "내가 어떤 작업을 걸어뒀더라?"를 떠올리려면 채팅 기록을 하나하나 거슬러 올라가야 했습니다. 이제는 그럴 필요 없이 대시보드 한 장으로 끝납니다. 작업을 잠시 멈추거나, 실행 주기를 바꾸거나, 아예 지우는 일도 페이지를 떠나지 않고 처리할 수 있습니다.

작은 변화처럼 보이지만, '예약 작업'이라는 기능을 **실제 일상에 들여놓기 위한 인프라**가 만들어졌다고 보는 편이 정확합니다.

### 리서치 작업 — 변화가 있을 때만 알린다

특히 눈여겨볼 부분은 **리서치 작업의 동작 방식**입니다. ChatGPT가 웹과 연결된 앱을 주기적으로 살피다가, 결과에 의미 있는 변화가 생긴 경우에만 알림을 띄웁니다.

예를 들어 경쟁사 가격 페이지, 자주 보는 기술 블로그의 새 글, 특정 키워드로 들어오는 뉴스 같은 것을 걸어두면, 매번 똑같은 결과를 반복해 보여주지 않고 **달라진 시점에만 보고**가 들어옵니다. 알림 피로도를 줄이면서 자동 모니터링 효과는 가져가는 방식입니다.

OpenAI는 이번 업데이트로 모든 작업이 더 빠르고 안정적으로 돌아간다고 설명합니다. 이는 곧 매번 결과를 확인하지 않아도 신뢰하고 맡겨둘 수 있는 수준을 노린다는 뜻입니다.

### 예약 옵션 — 시간 단위 + 시간대 단위

스케줄링 자체도 더 유연해졌습니다. 작업을 거는 방법이 두 가지로 정리됐습니다.

- **특정 시각 지정**: "매일 오전 8시 30분"처럼 정확한 시간 설정
- **시간대 지정**: 오전(morning), 오후(afternoon), 저녁(evening) 중 한 블록 선택

매일 같은 시간을 고집할 필요가 없고, "저녁 시간에 한 번만 정리해서 보여줘" 식의 자연스러운 요청을 그대로 일정으로 만들 수 있습니다.

다만 무한정 돌아가지는 않습니다. 작업당 실행 빈도는 **시간당 1회**가 상한이고, 사용자가 한동안 ChatGPT를 쓰지 않으면 작업은 **자동 일시정지**됩니다. 비용·서버 부담을 막는 안전장치이자, 방치된 작업이 의미 없이 돌아가는 것을 막기 위한 설계입니다.

### 누가 쓸 수 있나, 그리고 Pulse는 어떻게 되나

예약 작업은 **Plus, Pro, Business, Enterprise** 구독자에게 열려 있습니다. 동시에 활성 상태로 둘 수 있는 작업 수는 요금제마다 다릅니다. 무료 사용자는 이번 업데이트에 포함되지 않았습니다.

또 하나의 큰 변화는 **'Pulse' 기능의 종료**입니다. 기존에 매일 아침 요약을 받아 보던 Pulse는 이번에 예약 작업 안으로 흡수됩니다. 사용자는 자기가 원하는 주제·시간을 직접 예약 작업으로 잡으면 되고, 이미 만들어 둔 Pulse 사용 패턴도 같은 흐름으로 이전될 수 있습니다.

기능을 분리해 두기보다 **'예약 작업'이라는 단일 컨테이너**로 합치는 방향입니다. 사용자 입장에서는 어디서 어떤 알림을 켜뒀는지 한곳에서 추적할 수 있고, OpenAI 입장에서는 백엔드 인프라와 알림 정책을 일관되게 운영할 수 있습니다.

### 'AI 개인 비서'로 한 걸음 더

따로 보면 작은 UI 변경이지만, 흐름을 보면 의도가 분명합니다. 채팅창 안에 머물던 ChatGPT가 **일정·모니터링·알림까지 챙기는 상시 비서**로 영역을 넓히고 있습니다. 매번 명령을 다시 입력하지 않아도, 한 번 걸어둔 작업이 알아서 돌아가고 의미 있는 변화만 골라 알려주는 구조입니다.

같은 자리를 노리는 Google Gemini, Apple Intelligence, Microsoft Copilot이 모두 비슷한 방향으로 움직이고 있어, 이번 업데이트는 **'개인 비서 AI'의 다음 라운드**를 본격적으로 알리는 신호로 볼 수 있습니다.

## 실무자가 볼 핵심 포인트

1. **자동화 워크플로 재정비 기회**: 일정 단위로 ChatGPT를 호출해 정보를 모아오던 기존 자동화(예: cron + API 호출)는 일부 예약 작업으로 대체할 수 있습니다. 빈도 제한(시간당 1회)과 비활성 자동 일시정지를 감안해 무엇을 ChatGPT에 맡길지 다시 줄을 그어볼 시점입니다.

2. **모니터링 알림 설계 변화**: 리서치 작업이 '변화가 있을 때만 알림'을 보장한다면, 사내에서 운영하던 변동 모니터링(가격·문서·뉴스)을 ChatGPT 쪽으로 위임해볼 만합니다. 다만 알림 누락 시 책임 소재가 모호해질 수 있으니 핵심 지표는 자체 모니터링과 이중화하는 편이 안전합니다.

3. **Pulse 의존 워크플로 재검토**: Pulse로 매일 받던 요약·브리핑을 쓰고 있었다면, 이번 통합 시점에 같은 결과를 받아낼 수 있도록 예약 작업으로 옮겨두어야 합니다. 자동 마이그레이션이 매끄럽지 않을 가능성을 염두에 두고 한 번씩 점검이 필요합니다.

4. **요금제별 작업 수 한도 확인**: Plus·Pro·Business·Enterprise 사이에 동시 활성 작업 수가 다릅니다. 팀 단위로 도입할 때는 한 사람당 몇 개를 굴릴 수 있는지를 먼저 확인해야 라이선스 정책을 정할 수 있습니다.

5. **에이전트형 기능과의 합산 고려**: 예약 작업·Operator(웹 자동 조작)·Connectors(사내 앱 연결) 같은 기능이 동시에 진화 중입니다. 어느 하나만 따로 보면 작은 변화지만, 합치면 'ChatGPT가 알아서 일을 시작하고 끝내는' 그림이 되니, 도입 로드맵을 한 장으로 정리해두면 의사결정이 빨라집니다.
