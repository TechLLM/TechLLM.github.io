## 개요

앤트로픽이 새로 내놓은 **Claude Tag**는 슬랙 안에서 항상 켜져 있는 클로드입니다. 채널에 `@Claude`로 부르면 답을 주는 정도가 아니라, 대화 흐름을 계속 따라가면서 맥락을 쌓고, 필요할 때는 먼저 끼어들어 알려주는 동료 같은 존재로 설계됐습니다. 베타는 슬랙용 Claude Enterprise와 Claude Team 고객에게 먼저 열립니다. 겉으론 슬랙 봇 업그레이드처럼 보이지만, 속을 들여다보면 회사 내부의 일하는 방식과 지식을 한꺼번에 흡수하려는 더 큰 그림이 있습니다.

![슬랙에서 동료처럼 동작하는 클로드 태그](/images/anthropic-claude-tag-slack-ai-teammate/cover.png)

## 핵심 요약

- Claude Tag는 슬랙 채널에 상주하면서 대화를 따라가고, 멘션이나 작업 지시에 응답하는 **"항상 켜져 있는 클로드"** 입니다.
- 기존 `@Claude` DM, 채널 멘션, Claude Code 슬랙 연동을 이어받으면서 **지속 맥락과 기억**을 새로 얹었습니다.
- 채널 단위로 **하나의 클로드 정체성**을 공유합니다. 이전 사람이 멈춘 지점에서 누구든 이어 받을 수 있습니다.
- 관리자가 **접근 가능한 채널·도구·정보를 명시적으로 지정**합니다. 법무용 클로드가 엔지니어링 채널에 기억을 흘리는 일은 막힙니다.
- 작업이 떨어지면 **단계로 쪼개 진행**하고, 결과물을 스레드에 올려 공개합니다.
- **앰비언트 모드**가 있어 누가 부르지 않아도 먼저 끼어듭니다. 잊힌 스레드나 다른 채널 소식을 알아서 끌어다 알려줍니다.
- 마이크로소프트 Graph(Copilot, Work IQ), 스노우플레이크, 데이터브릭스, Glean과 같은 **기업 맥락 레이어** 경쟁의 한복판에 들어갔습니다.

## 본문

### 기존 클로드 슬랙 연동과 무엇이 다른가

앤트로픽은 이미 슬랙 안에서 클로드와 대화할 수 있는 방법을 여러 갈래로 풀어놨습니다. DM으로 `@Claude`에게 묻거나, 채널에서 태그해 즉석 도움을 받는 식이죠. Claude Code 연동을 쓰면 채널 멘션에서 시작한 코딩 요청이 웹에서 본격적인 코딩 세션으로 이어지고, 결과는 다시 스레드에 올라옵니다.

Claude Tag가 한 발 더 나간 지점은 **맥락 보존**입니다. 기존 도구는 묻고 답하면 거기서 끝났지만, Claude Tag는 채널을 떠나지 않고 대화 흐름을 그대로 따라갑니다. 앤트로픽은 "채널을 따라가는 동안 클로드는 그 일에 대해 더 많이 배워간다"고 설명합니다. 관리자가 권한을 주면 다른 채널에서 사실관계도 끌어옵니다.

### 채널마다 한 명의 클로드, 그리고 그 효과

가장 눈에 띄는 설계는 **채널당 하나의 클로드 정체성**입니다. 그 방에 있는 모두가 같은 클로드를 보고, 클로드가 지금까지 무엇을 했는지 그대로 들여다볼 수 있습니다. 누군가 잠시 자리를 비웠다가 돌아와도 클로드가 멈춰 있던 지점부터 다시 시작할 수 있다는 뜻입니다. 인수인계가 없는 인수인계입니다.

대신 보안 측면에서 분리는 단단하게 했습니다. 어떤 도구를 쓸 수 있는지, 어떤 정보를 읽을 수 있는지, 어떤 채널에 들어갈 수 있는지를 관리자가 채널 단위로 정합니다. 법무 채널 클로드가 엔지니어링 채널로 기억을 흘려보내지 못하도록 정체성 자체가 채널 범위에 묶입니다. 기업이 가장 걱정하는 정보 누수 시나리오를 정면으로 막아 둔 셈입니다.

### 가만히 있지 않는 AI, 앰비언트 모드

Claude Tag의 또 다른 무기는 **앰비언트 모드**입니다. 사람이 부르지 않아도 클로드가 알아서 채팅에 끼어듭니다. 팀 진행 상황을 정리해주고, 조직 다른 곳에서 일어난 일을 알려주고, 며칠째 답변이 없는 스레드나 잊힌 작업을 끌어내 다시 챙기게 합니다.

앤트로픽은 "공개적으로 결과물을 만들어내고, 이전보다 훨씬 풍부한 맥락을 가진 진짜 동료와 일하는 느낌"이라고 표현합니다. 슬랙 봇이 명령을 기다리는 자판기였다면, Claude Tag는 회의실 구석에서 모든 대화를 듣고 있다가 필요할 때 손을 드는 신입에 가깝습니다. 이 비유가 누군가에게는 든든하게, 누군가에게는 부담스럽게 들릴 수 있습니다.

### 진짜 노림수: 회사 맥락이라는 자산

기능 한 줄로 보면 "슬랙에 더 똑똑한 봇이 들어왔다" 정도로 보입니다. 하지만 시장 흐름과 같이 두고 보면 그림이 달라집니다. 마이크로소프트는 Graph를 만들어 Copilot과 Work IQ로 표면화하고 있고, 스노우플레이크와 데이터브릭스는 자기 플랫폼이 회사의 암묵지를 담는 백엔드라고 주장합니다. Glean은 모델과 회사 데이터 사이에 끼어 있는 지능 레이어를 짓고 있습니다.

모두가 같은 자리를 노립니다. **모델이 아니라 회사 맥락**입니다. 어떤 LLM을 쓰든 결국 그 회사의 일하는 방식, 의사결정 패턴, 누가 무엇을 알고 있는지를 가장 잘 아는 쪽이 다음 라운드의 승자가 됩니다. 앤트로픽이 슬랙 한가운데에 항상 켜져 있는 클로드를 심겠다는 건, 모델 성능 경쟁이 아니라 그 다음 단계의 게임을 시작하겠다는 신호로 읽힙니다.

## 실무자가 볼 핵심 포인트

- **도입 결정자라면**: Claude Tag는 슬랙 자동화 도구가 아니라 사내 맥락 저장소에 가깝습니다. 어떤 채널에 들이고 어떤 채널에서 막을지를 처음부터 보안 정책으로 설계해야 합니다. "일단 다 켜고 보자"는 위험합니다.
- **IT/보안 담당자라면**: 채널 단위 권한 분리, 도구 접근 화이트리스트, 다른 채널 읽기 권한 부여 정책을 먼저 정리하세요. 정체성이 채널에 묶인다는 설계 의도를 깨면 사고가 납니다.
- **PM·팀 리드라면**: 앰비언트 모드는 잊힌 스레드와 작업을 끌어내는 데 강합니다. 다만 끼어드는 빈도가 늘면 노이즈가 됩니다. 채널별로 어느 정도까지 자발적으로 말하게 할지 가이드라인이 필요합니다.
- **개발자라면**: 기존 `@Claude` 멘션이나 Claude Code 슬랙 연동을 쓰던 워크플로는 Claude Tag로 자연스럽게 흡수됩니다. 채널이 동일한 클로드 정체성을 공유한다는 점을 활용해, 코드 리뷰 스레드의 컨텍스트가 다음 작업에도 살아남도록 채널 구조를 다시 짜는 게 효율적입니다.
- **경쟁 관점**: 마이크로소프트 Graph, Glean, 데이터브릭스/스노우플레이크와 비교할 때 앤트로픽의 강점은 **모델 자체의 추론 품질 + 슬랙 표면 장악**입니다. 이미 슬랙 중심으로 일하는 회사라면 가장 마찰이 적은 진입로일 수 있습니다.
