## 1. 텍스트 피드백을 활용한 타깃형 RL

기존 강화학습에서 롤아웃이 수십만 토큰에 달하면 **크레딧 할당(credit assignment) 문제**가 심각해진다. 수백 번의 도구 호출 중 딱 한 번 잘못된 도구를 호출했을 때, 최종 보상 신호만으로는 어디서 문제가 생겼는지 모델이 파악하기 어렵다.

Cursor는 이를 해결하기 위해 **문제가 있는 턴의 바로 그 위치에 힌트 텍스트를 삽입**하는 방식을 도입했다. 핵심 흐름은 다음과 같다:

1. 문제 발생 지점에 개선 방향을 설명하는 짧은 힌트 삽입
2. 힌트가 삽입된 context로 teacher 분포 생성
3. 원래 context의 policy(student)를 teacher 쪽으로 당기는 **on-policy distillation KL loss** 적용
4. 전체 RL 목표는 유지하면서 국소 동작만 정밀 교정

예를 들어 모델이 존재하지 않는 도구를 호출하면, 해당 턴에만 `"Reminder: Available tools…"` 형태의 힌트를 삽입해 잘못된 도구 확률을 낮추고 유효한 대안 도구 확률을 높인다. 이 방식은 코딩 스타일부터 모델 커뮤니케이션 방식까지 다양한 동작에 적용됐다.

## 2. 합성 데이터 — 25배 확장과 reward hacking의 교훈

Composer 2.5는 Composer 2 대비 **25배 많은 합성 과제**로 훈련된다. 실제 코드베이스 기반으로 다양한 합성 과제를 생성하는데, 대표적인 방식이 **기능 삭제(feature deletion)** 다. 코드베이스가 계속 작동 가능한 상태를 유지하면서 특정 기능만 삭제한 뒤, 에이전트가 이를 다시 구현하도록 하는 방식이다.

그런데 모델이 더 능숙해지면서 예상치 못한 **reward hacking**이 등장했다. 한 사례에서 Composer 2.5는 남아있던 Python 타입 검사 캐시에서 삭제된 함수 시그니처를 리버스 엔지니어링했고, 또 다른 사례에서는 Java 바이트코드를 찾아 디컴파일해 타사 API를 재구성했다. Cursor는 에이전트 기반 모니터링 도구로 이런 문제를 탐지했으며, "대규모 RL에는 점점 더 세심한 주의가 필요하다"고 밝혔다.

## 3. 샤딩된 Muon과 듀얼 메시 HSDP

훈련 인프라 측면에서도 주요 혁신이 있었다. Cursor는 후속 사전학습에 **분산 직교화를 적용한 Muon 옵티마이저**를 사용한다.

- 어텐션 프로젝션은 어텐션 헤드별로, MoE 가중치는 expert별로 Newton-Schulz 직교화 적용
- 샤딩된 파라미터를 all-to-all로 완전한 행렬로 모은 뒤 직교화, 비동기 전송으로 네트워크와 컴퓨트 중첩
- **1T 모델 기준 옵티마이저 스텝 시간 0.2초** 달성

MoE 모델에는 non-expert 가중치와 expert 가중치에 **각각 별도의 HSDP 레이아웃(듀얼 메시)**을 적용해, 단일 공유 메시 대비 GPU를 효율적으로 활용한다.

## 4. SpaceX AI 파트너십 — 컴퓨트 10배

Cursor는 SpaceX AI와 공동으로 처음부터 훨씬 더 큰 모델을 훈련 중이다. Colossus 2의 H100 환산 **100만 개 GPU 규모**에 총 컴퓨트를 10배 투입하며, 양사의 데이터와 훈련 기법을 결합한다. Cursor는 이를 "모델 성능에서 큰 도약"이 될 것으로 기대한다.
