YC 지원 스타트업 Lilac이 MiniMax와 공식 파트너십을 체결하고 M2.7 모델을 상업적으로 제공한다. 오픈 웨이트 모델임에도 비상업적 라이선스 제한이 걸려 있던 M2.7을 기업 환경에서 바로 쓸 수 있는 경로가 열린 것이다.

## 핵심 요약

- Lilac(YC)이 MiniMax와 파트너십으로 **M2.7 모델을 상업적으로 제공** 시작
- 입력 $0.30 / 캐시 읽기 $0.055 / 출력 $1.20 (백만 토큰 기준)
- 160-way 동시성에서 **사용자당 60 tok/s 지속**, 요청 성공률 100%
- 도구 호출 일치율 98.80%, 스키마 정확도 99.80%
- OpenAI 호환 API로 즉시 사용 가능

## M2.7이란?

MiniMax M2.7은 전문 소프트웨어 엔지니어링, 장기 프로젝트, 도구 집약적 에이전트를 목표로 설계된 오픈 웨이트 모델이다. 코딩 에이전트, 긴 컨텍스트 처리, 신뢰할 수 있는 도구 호출 세 가지가 핵심 설계 목표다.

문제는 라이선스였다. 공개 가중치는 **비상업적 사용만 허용**되어 있어, 기업이 직접 M2.7을 프로덕션에 투입하려면 별도 상업적 라이선스 계약이 필요했다. 이번 Lilac 파트너십은 이 진입 장벽을 낮추는 역할을 한다.

## 왜 Lilac인가?

Lilac은 대규모 모델 배포를 팀 자체적으로 운영하지 않아도 되도록 **호스팅 엔드포인트, GPU 용량 라우팅, 표준 API 인터페이스**를 제공하는 인프라 레이어다. 고객 입장에서는 GPU 클러스터 운영 부담 없이 M2.7을 바로 API로 호출할 수 있다.

상업적 측면에서는 Lilac과의 파트너십 계약을 통해 M2.7의 상업적 이용 경로가 확보된다. 즉 Lilac을 통해 M2.7을 사용하면 MiniMax 측 상업 라이선스를 별도 협상할 필요가 없다.

## 벤치마크 — 실제 수치

Lilac이 공개한 AIPerf 스트레스 테스트와 MiniMax Provider Verifier 결과다.

**AIPerf 스트레스 테스트**

| 지표 | 결과 |
|------|------|
| 집계 출력 처리량 | 15,544.8 output tok/s |
| 사용자당 처리량 | 지속적으로 60 tok/s/user |
| TTFT P50 | 1.2s |
| TTFT P90 | 3.3s |
| 요청 성공률 | 100.0% |

160-way 동시성 — 160명이 동시에 요청을 보내는 상황 — 에서도 사용자당 60 tok/s를 유지했다는 점이 눈에 띈다. 분당 3,600 토큰 이상이면 일반적인 코드 리뷰나 에이전트 추론에 충분한 속도다.

**MiniMax Provider Verifier**

| 지표 | 결과 |
|------|------|
| 성공한 요청 | 1,020 / 1,020 |
| 쿼리 성공률 | 100% |
| 도구 호출 일치율 | 98.80% |
| 스키마 정확도 | 99.80% |

도구 호출 일치율 98.80%는 에이전트 워크플로에서 중요한 수치다. 함수 호출이 자주 실패하거나 스키마를 어기면 에이전트 파이프라인 전체가 무너지기 때문이다.

## 가격 구조

| 구분 | 가격 (백만 토큰당) |
|------|-----------------|
| 입력 | $0.30 |
| 캐시 읽기 | $0.055 |
| 출력 | $1.20 |

캐시 읽기가 입력 대비 약 1/5 수준이라 반복적인 시스템 프롬프트나 긴 컨텍스트를 자주 재사용하는 에이전트라면 비용 절감 여지가 있다.

## 사용 방법

OpenAI 호환 API라 기존 코드에서 `base_url`과 `model` 두 줄만 바꾸면 된다.

```python
from openai import OpenAI

client = OpenAI(
    base_url="https://api.getlilac.com/v1",
    api_key="lilac_sk_...",
)

response = client.chat.completions.create(
    model="minimaxai/minimax-m2.7",
    messages=[
        {"role": "user", "content": "Review this pull request for production risks."},
    ],
)
```

## 실무자가 볼 핵심 포인트

- **라이선스 확인 필수**: M2.7 공개 가중치는 비상업적 전용이다. 상업 프로덕션 배포는 반드시 Lilac 같은 공인 파트너를 통하거나 MiniMax와 직접 계약해야 한다.
- **에이전트 워크플로 적합도**: 도구 호출 일치율 98.80%는 준수한 수준이지만, 미션 크리티컬 파이프라인이라면 자체 환경에서 도구 호출 실패율을 별도로 검증할 것.
- **GPU 운영 부담 제거**: 직접 M2.7을 온프레미스 배포하면 GPU 클러스터 운영·스케일링이 필요하다. Lilac API 방식이면 이 부담을 즉시 제거할 수 있다.
- **캐시 전략**: 입력 캐시 읽기 단가가 입력의 18% 수준이므로, 동일 시스템 프롬프트를 반복 사용하는 에이전트는 캐싱 설계로 비용을 크게 절감할 수 있다.
- **경쟁 비교 포인트**: $0.30/$1.20 (입출력) 가격대는 GPT-4o mini보다 비싸고 Claude Sonnet 계열보다 저렴한 중간대다. 코딩 에이전트 특화 성능이 실제 태스크에서 비용 차이를 정당화하는지 사내 벤치마크로 확인해볼 것.
