# TraceMint — Positive Trace Mining for Agent Improvement

Auto-generated and experimental; review outputs before using them in production.

## Overview

TraceMint converts successful AI agent execution logs into clean, reusable positive trace datasets. It focuses on runs that already passed an external grader or verification process, then normalizes tool calls, evidence, edits, state changes, and final artifacts into deterministic JSONL and Markdown corpora.

Use it when successful traces are more actionable than manually labeling every failure mode. The bundled CLI is intentionally small and self-contained so it can be adapted to local agent log formats without external services.

## When to use

- Use when you have JSONL agent execution logs with grader, verifier, or status fields that mark successful runs.
- Use when you need positive trace mining for prompt tuning, few-shot examples, tool policy refinement, retrieval policy analysis, or regression-aware agent development.
- Use when you want redacted, human-reviewable Markdown alongside machine-readable JSONL.
- Use when you need a deterministic sample implementation that can run without API keys.
- When NOT to use: do not use this as a replacement for a security audit, full failure analysis, or production-grade data governance pipeline.

## Workflow

1. Collect agent execution logs as JSONL, one run per line.
2. Ensure passing runs include a grader-like signal such as `{"grader": {"passed": true}}`, `{"grade": {"verified": true}}`, or `{"status": "passed"}`.
3. Run `python scripts/run.py --input path/to/logs.jsonl --out-jsonl build/traces.jsonl --out-md build/traces.md`.
4. Add extra redaction rules with `--redact-regex` or the `TRACEMINT_REDACT_PATTERNS` environment variable when logs contain internal names or identifiers.
5. Review the Markdown output for quality and sensitive content.
6. Use the JSONL output as a positive trace corpus for prompt examples, tuning preparation, policy audits, retrieval benchmarks, or regression documentation.

## Inputs & Outputs

Input is a JSONL file with one JSON object per line. Each run may contain:

- `run_id` or `id`: stable run identifier.
- `task`: task text or summary.
- `grader`, `grade`, `verifier`, `evaluation`, or `status`: pass/fail signal.
- `events`: ordered execution events.
- `retrieved_evidence`, `artifacts`, `file_diffs`, or `state_changes`: optional top-level collections.

Recognized event types include `tool_call`, `retrieval`, `evidence`, `file_edit`, `state`, `state_change`, `final_artifact`, and `artifact`. Unknown event types are preserved as compact generic events.

Machine-readable output is JSONL, one positive trace per line, with this shape:

```json
{
  "trace_id": "string",
  "task": "string",
  "score": 1.0,
  "event_count": 6,
  "tools": ["string"],
  "evidence": [{"source": "string", "snippet": "string", "metadata": {}}],
  "artifacts": [{"kind": "string", "summary": "string"}],
  "events": [{"index": 0, "type": "tool_call"}]
}
```

Markdown output uses this shape:

```markdown
# TraceMint Positive Trace Dataset

Generated by TraceMint.

## Trace <trace_id>
- Task: <task>
- Score: <score>
- Events: <event_count>
- Tools: <comma-separated tool names or none>

### Evidence
1. `<source>` - <snippet>

### Event Sequence
1. <event type>: <event label>

### Artifacts
- <kind>: <summary>
```

## Installation

Copy this skill directory into your skills folder, or run it directly from this directory:

```bash
python scripts/run.py --help
python scripts/test.py
```

No package install is required because the reference CLI uses only the Python standard library.

## Usage

Show help:

```bash
python scripts/run.py --help
```

Run the built-in self-test sample:

```bash
python scripts/run.py --selftest
```

Mine a JSONL agent log:

```bash
python scripts/run.py \
  --input examples/sample.jsonl \
  --out-jsonl build/traces.jsonl \
  --out-md build/traces.md
```

Add redaction rules:

```bash
python scripts/run.py \
  --input examples/sample.jsonl \
  --out-jsonl build/traces.jsonl \
  --out-md build/traces.md \
  --redact-regex "internal-project-[0-9]+"
```

Use environment redaction patterns:

```bash
TRACEMINT_REDACT_PATTERNS='["customer-[0-9]+", "private-team-name"]' \
python scripts/run.py --input examples/sample.jsonl --out-jsonl build/traces.jsonl
```

## Example

Command:

```bash
python scripts/run.py \
  --input examples/sample.jsonl \
  --out-jsonl build/traces.jsonl \
  --out-md build/traces.md
```

Expected summary:

```json
{"input_runs": 2, "markdown_path": "build/traces.md", "output_path": "build/traces.jsonl", "passing_traces": 1, "status": "ok"}
```

The JSONL output should match `examples/expected.jsonl`, and the Markdown output should match `examples/expected.md`.

## Limitations

- The reference parser supports common event shapes, not every possible agent telemetry schema.
- Redaction is best-effort and should be reviewed before sharing data outside a trusted environment.
- Passing-run detection depends on explicit grader, verifier, or status fields.
- The CLI extracts positive traces only; it does not train models or call external APIs.
