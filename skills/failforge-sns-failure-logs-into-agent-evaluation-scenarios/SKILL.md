# FailForge - SNS Failure Logs into Agent Evaluation Scenarios

Auto-generated & experimental skill for turning social publishing failures into reusable agent evaluation cases.

## Overview

FailForge helps convert social network publishing logs into structured failure reports and YAML scenarios for agent regression suites. It is designed for messy execution traces from publishing pipelines, including expired credentials, API errors, rate limits, media container delays, validation failures, network timeouts, and dry-run traces.

The bundled CLI uses deterministic standard-library parsing, regular expressions, and rule-based classification. It does not require an LLM call, an API key, or any private service.

## When to use

Use this skill when you need to:

- Turn social publishing execution logs into agent evaluation scenarios.
- Classify failures from Instagram, Facebook, TikTok, LinkedIn, X, YouTube, or generic SNS publishing jobs.
- Build regression cases that test whether an agent can diagnose failures, choose safe recovery actions, and avoid unsafe retries.
- Redact sensitive-looking tokens, account identifiers, URLs, and headers before logs become evaluation artifacts.

Trigger keywords: `failforge`, `SNS failure logs`, `social publishing logs`, `agent evaluation scenarios`, `expired credentials`, `rate limits`, `media containers`, `dry-run traces`.

## Installation

Copy this skill directory into a Claude Code or OpenClaw compatible skills folder.

Example:

```bash
mkdir -p skills
cp -R failforge-sns-failure-logs-into-agent-evaluation-scenarios skills/
```

No Python packages are required. The CLI uses only the Python standard library.

## Usage

Run with built-in sample data:

```bash
python scripts/run.py
```

Parse the included example log:

```bash
python scripts/run.py examples/social_publish_failures.log
```

Write machine-readable outputs:

```bash
python scripts/run.py examples/social_publish_failures.log --output-dir out
```

Print only normalized JSON:

```bash
python scripts/run.py examples/social_publish_failures.log --format json
```

Print only scenario YAML:

```bash
python scripts/run.py examples/social_publish_failures.log --format yaml
```

Read from stdin:

```bash
cat examples/social_publish_failures.log | python scripts/run.py --format yaml
```

## Example

Input log line:

```text
2026-06-30T10:20:11Z platform=tiktok request_id=req_sample_rate_002 status=429 error="Rate limit exceeded; retry after 60 seconds"
```

Generated scenario excerpt:

```yaml
scenarios:
  -
    id: "failforge-001"
    failure_family: "rate_limit"
    expected_diagnosis: "The publish attempt was throttled by the platform or API gateway."
```

## Limitations

- Classification is rule-based and intentionally conservative.
- YAML output is generated by a small built-in serializer that covers the simple scenario schema used by this skill.
- The CLI does not call social platform APIs or validate live credentials.
- Redaction catches common sensitive patterns but should not be treated as a complete data-loss-prevention system.
